## 1. Summary

SnapInsight 是一个面向 Chrome 桌面浏览器的本地辅助理解插件。用户在网页中选中短文本后，插件在选区附近显示悬浮图标；用户悬浮图标后查看简短解释，点击“查看更多”后在同一卡片内展开详细解释。

解释内容由本机 Python 服务调用本地 Ollama 模型生成。该产品用于在尽量不打断阅读流程的前提下，为用户提供本地化、分层式的概念解释能力。

## 2. Problem Statement

- 用户在阅读网页内容时，经常需要临时理解词语、短语、术语、人名或地名。
- 传统做法通常需要切换页面、手动搜索或打开新标签页，容易打断阅读流程。
- 用户希望在当前页面内获取“简短解释 + 详细解释”的分层信息，而不是离开原始阅读上下文。
- 对于注重隐私或偏好本地模型的用户，解释请求应仅发送到本机服务。

## 3. Goals and Non-Goals

### 3.1 Goals

- 帮助用户在阅读网页内容时快速理解短文本概念。
- 提供“简短解释 + 详细解释”的分层信息获取方式，降低解释行为对阅读流程的干扰。
- 通过本地 Python 服务与本机 Ollama 模型生成内容，降低隐私暴露风险。
- 为后续扩展到更广泛用户使用预留产品基础。

### 3.2 Non-Goals

- 长文本解释
- 历史记录
- 收藏功能
- 多语言界面
- 账号体系
- 云端模型接入
- 同步功能
- 复杂富文本排版
- PDF 页面支持
- `chrome://` 页面支持
- Chrome Web Store 页面支持
- 扩展自身页面支持
- 富文本编辑器支持
- `iframe` 内容支持

## 4. Target Users and Core Scenarios

### 4.1 Target Users

- 首阶段主要面向项目作者个人使用。
- 后续可扩展为面向更广泛 Chrome 用户发布。

### 4.2 Core Scenarios

- 用户在阅读网页内容时，希望快速理解某个概念。
- 用户选中词语、短语、术语、人名或地名后，希望立即获得简短解释。
- 当简短解释不足以满足需要时，用户希望继续查看更完整的定义、背景、使用场景和示例。

## 5. Scope

### 5.1 In Scope

- 支持 Chrome 桌面版浏览器。
- 支持用户手动选中的短文本。
- 支持的选中文本类型包括：
  - 单词
  - 短语
  - 术语
  - 人名
  - 地名
- 选中文本长度限制为 `1-20` 个字或词。
- 超过长度上限时，不显示悬浮图标。
- 选中后在选区附近立即显示悬浮图标。
- 用户悬浮图标时显示解释卡片。
- 卡片内提供“查看更多”入口，并在同一卡片内展开详细解释。
- 插件通过本地 Python 服务调用本机 Ollama 模型生成解释内容。
- 插件支持读取本机 Ollama 模型列表，并通过下拉框切换模型。

### 5.2 Support Boundary

- 支持普通网页中的文本选中。
- 支持 `input` 和 `textarea` 中的文本选中。
- 不支持 PDF 页面中的文本选中。
- 不支持 `chrome://` 等 Chrome 内部页面。
- 不支持 Chrome Web Store 页面。
- 不支持扩展自身的 popup、options 等页面。
- 不支持富文本编辑器中的文本选中。
- 不支持 `iframe` 内容中的文本选中。

### 5.3 Out of Scope

- 长文本解释
- 富文本卡片和复杂排版
- 历史记录和收藏
- 多语言 UI
- 账号与同步能力
- 云端模型接入

## 6. User Flows

### 6.1 Primary Flow

1. 用户在网页中选中一段短文本。
2. 插件在选区附近立即显示悬浮图标。
3. 用户将鼠标悬浮到图标上。
4. 插件展示解释卡片，并请求该文本的简短解释。
5. 卡片内展示加载状态，并以流式方式逐步显示简短解释内容。
6. 解释卡片保持显示。
7. 用户点击“查看更多”。
8. 插件在当前卡片区域内展开详细解释，并展示加载状态。
9. 详细解释内容以流式方式逐步显示。
10. 用户点击页面空白处，或点击卡片关闭按钮，结束本次查看。

### 6.2 Alternative Flows

- 如果卡片已打开，用户再次选中新的短文本时，插件关闭旧卡片并创建新卡片。
- 如果简短解释已成功显示，用户可以不展开详细解释，直接关闭卡片。
- 如果模型已完成本地持久化选择，后续请求直接使用已选模型。

### 6.3 Failure Flows

- 当本地 Python 服务未运行时，插件在卡片内显示友好错误提示。
- 当 Ollama 正常运行但没有可用模型时，插件显示友好错误提示，并引导用户配置模型。
- 当模型列表无法加载且用户尚未完成首次模型选择时，插件阻止解释请求并提示用户先完成模型配置。
- 当简短解释成功但详细解释失败时，插件保留简短解释内容，并在详情区域显示错误提示，同时提供重试入口。
- 当用户在旧请求仍在加载时选中新的文本，插件取消旧请求并切换到新的文本。
- 当详细解释仍在加载中而用户关闭卡片时，插件尽可能取消对应请求。

## 7. Functional Requirements

### 7.1 Selection Detection

- 插件应检测网页中的文本选中行为。
- 仅当选中文本非空且满足长度限制时，才触发悬浮图标显示。
- 对超过长度上限的文本，不显示悬浮图标。

### 7.2 Floating Trigger

- 图标应在用户选中短文本后立即出现。
- 图标位置应尽量靠近选区，避免明显遮挡正文。
- 图标样式应清晰、轻量，并适合在大多数网页背景下识别。

### 7.3 Short Explanation

- 用户悬浮图标时，应展示解释卡片。
- 简短解释输出长度应限制在 `120` 字以内。
- 输出语言以中文为主，必要时可保留原文术语。
- 简短解释应直观、易懂，适合非技术用户阅读。
- 简短解释请求应支持卡片内加载状态展示。
- 简短解释内容应支持流式显示。

### 7.4 Detailed Explanation

- 详细解释由用户点击“查看更多”后触发。
- 首次展示卡片时，仅请求简短解释。
- 点击“查看更多”后，再单独请求详细解释。
- 详细解释请求应支持卡片内加载状态展示。
- 详细解释内容应支持流式显示。
- 详细解释应优先覆盖以下信息：
  - 更完整的定义
  - 背景说明
  - 使用场景
  - 举例
- 当详细解释失败时，应仅在详情区域展示错误提示，并支持用户重试。

### 7.5 Model Selection

- 插件应从本机 Ollama 服务读取可用模型列表。
- 用户首次使用时必须显式选择模型后，才能发起解释请求。
- 用户选择的模型应在本地持久化。
- 用户应可通过下拉框选择当前使用的模型。
- 如果之前选择的模型不再可用，应要求用户重新选择模型。
- 当本机无可用模型时，应提示用户先配置模型。

### 7.6 Error Handling

- 当本地服务不可用、模型调用失败或请求异常时，插件应向用户反馈明确提示。
- 提示文案应优雅、礼貌，并对非技术用户友好。
- 异常提示应避免暴露不必要的技术细节。
- 当模型列表无法加载且用户尚未完成首次模型选择时，应阻止解释请求并提示用户完成模型配置。

## 8. Interaction Rules

- 解释卡片在出现后应保持显示。
- 鼠标从图标移动到卡片时，卡片应继续保持显示。
- 鼠标移开时，卡片不自动关闭。
- 用户可通过点击页面空白处关闭卡片。
- 用户可通过点击卡片上的关闭按钮关闭卡片。
- 详细解释不采用独立大浮层，而是在当前卡片区域内展开或收起。
- 如果卡片已存在且用户选中新的文本，插件关闭旧卡片并创建新卡片。
- 当详细内容已展开时，点击关闭按钮应直接关闭整个卡片，而不是先收起详情。
- 简短解释和详细解释均应具备明确的加载状态。

## 9. Non-Functional Requirements

### 9.1 Usability

- 插件不影响用户正常选中文本的行为。
- 插件交互应尽量轻量，不明显打断阅读流程。
- 卡片显示和关闭行为应可预期、稳定。

### 9.2 Performance

- 本文档不规定具体响应时间指标。
- 当请求耗时较长时，应提供清晰但轻量的加载反馈。
- 当请求失败或超时时，应及时展示错误提示。

### 9.3 Privacy and Security

- 用户选中的文本只允许发送到本机服务，不允许发送到外网。
- 产品不记录查询历史。
- 产品不保存用户选中文本。

### 9.4 Compatibility

- 兼容范围限于 Chrome 桌面浏览器中的普通网页文本场景，以及 `input` / `textarea` 选中文本场景。
- PDF、Chrome 内部页面、扩展自身页面、富文本编辑器和 `iframe` 内容不在兼容范围内。

## 10. Acceptance Criteria

- 用户选中 `1-20` 个字或词的短文本时，插件可在选区附近显示图标。
- 插件可在普通网页文本和 `input` / `textarea` 场景中工作。
- 用户悬浮图标后，可查看 `120` 字以内的简短解释。
- 简短解释和详细解释均具备卡片内加载状态。
- 简短解释和详细解释均支持流式显示。
- 用户点击“查看更多”后，可在当前卡片区域内展开详细解释。
- 用户可通过点击页面空白处或点击关闭按钮关闭卡片。
- 当用户在加载过程中选中新词时，旧请求会被取消并切换到新的文本。
- 插件首次使用时要求用户显式选择模型。
- 插件可从本机 Ollama 获取模型列表，并支持下拉选择和本地持久化当前模型。
- 当已选模型失效时，插件要求用户重新选择模型。
- 当本地服务异常、无模型可用或详细解释失败时，插件可提供友好提示，并在适用场景下提供重试能力。
- 插件不发送选中文本到外网，不记录查询历史，不保存选中文本。

## 11. Dependencies and Assumptions

### 11.1 Dependencies

- 用户本机已安装 Chrome 浏览器。
- 用户本机已安装并启动 Ollama 服务。
- 本地 Python 服务已启动并可被插件访问。

### 11.2 Assumptions

- 用户本机存在至少一个可用 Ollama 模型，或可以在首次使用时完成模型配置。
- 后续技术设计将提供模型列表获取、流式响应和请求取消的具体实现方式。
